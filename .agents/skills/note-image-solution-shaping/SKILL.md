# メモ画像ソリューション整形

## 目的

手書きメモや資料写真を使うアイデアを、AI らしい変換が見え、初見で価値が伝わるデモ形に素早く整える。

## 最初に決めること

1. どんなメモ画像を入力にするのか。
2. AI が読み取る情報は何か。
3. AI が整理、変換、判断する結果は何か。
4. ユーザーが出力を見てすぐできる行動は何か。

## 強い形

- 雑なメモが、きれいな構造に変わる。
- 書いた本人以外にも意味が伝わる状態になる。
- AI が抜け漏れ、矛盾、優先度、次の作業を見つける。
- 結果が要約、タスク、分類、表、スコア、アクションに分かれている。
- 入力画像と変換後の結果を並べることで、Before/After が一目で分かる。

## 整形手順

1. アイデアを一文にする。
   例: 「[メモの種類] を撮ると、AI が [読み取り内容] を整理して、[使える出力] に変換する」
2. 入力画像の前提を決める。
   手書き、ホワイトボード、付箋、会議メモ、授業ノート、企画ラフ、チェックリストなど。
3. 出力の型を 1 つ選ぶ。
   タスク化、議事録化、優先順位付け、リスク抽出、計画化、比較表化、改善提案。
4. UI は Before/After を中心にする。
   左に画像、右に構造化結果。モバイルでは画像、結果の順に縦並び。
5. 読み取り失敗に備える。
   サンプル画像、手入力補正欄、モック結果を用意する。

## 判断基準

- メモ画像である必然性があるか。
- 入力の雑さが、出力の価値につながっているか。
- AI が単なる文字起こし以上の判断をしているか。
- 初見の審査員が Before/After の差をすぐ理解できるか。
- テーマに応じて、メモの種類と出力型を差し替えられるか。

## 避けること

- ただ OCR するだけの案。
- 出力が長い文章だけの案。
- 読み取り精度に依存しすぎる案。
- メモの内容とユーザーの次の行動がつながらない案。
- 画像がなくても同じ価値になる案。

## 最終出力

当日の相談では、次の形で返す。

- 一文コンセプト。
- 入力するメモ画像の種類。
- AI が読み取ること。
- AI が変換する出力。
- 画面構成。
- 3 分デモ手順。
- 実装を軽くする逃げ道。
